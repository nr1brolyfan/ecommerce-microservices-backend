Programowanie aplikacji internetowych opartych o mikrousługi
Specyfikacja wymagań projektowych

Cel
Zaprojektować i zrealizować backend w architekturze mikro-usługowej dla wybranego systemu (przykład: system e-commerce). Celem jest praktyczne przećwiczenie projektowania usług, ich komunikacji oraz dokumentowania decyzji projektowych. Frontend nie jest wymagany (opcjonalny przy dużym zespole).

Zakres
Obowiązkowo: backend składający się z wielu mikro-usług, komunikujących się przez REST/HTTP.
Przykładowy system e-commerce (zespół 2-3 osobowy):
Produkty (katalog, kategorie)
Użytkownicy + uwierzytelnianie (rejestracja, logowanie – może być prosty wariant)
Koszyk i zamówienia
Opinie/oceny/komentarze
Liczebność usług: orientacyjnie ~2 mikro-usługi na osobę w zespole.
Brak wymagań na UI; testy funkcjonalności za pomocą Postman/cURL.
Technologia
Typescript/Node.js
Do oddania
Kod źródłowy wszystkich mikro-usług w jednym ZIP (bez binariów).
Dokumentacja projektu (PDF), zawierająca przykładowo:
Opis domeny i celów projektu.
Architekturę: diagram komponentów (mikro-usługi, ich odpowiedzialności, bazy danych).
Specyfikacje API (endpointy, przykładowe request/response).
Konfigurację i instrukcję uruchomienia krok-po-kroku (lokalnie).
Wyniki testów: kolekcje Postman lub zrzuty ekranu z Postman/cURL (dla kluczowych endpointów).
Podział pracy: kto za co odpowiadał (na poziomie zadań/usług).
